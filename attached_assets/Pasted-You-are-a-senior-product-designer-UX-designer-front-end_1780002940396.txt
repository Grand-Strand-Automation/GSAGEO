## Calculator formulas and scoring logic

Implement real working formulas in reusable utility files.

### A. Current monthly spend
Calculate:
- managed support
- Microsoft 365
- email security
- endpoint security
- backup and recovery
- network or firewall support
- other recurring information technology tools
- internal admin overhead

Formula:
currentMonthlySpend =
managedSupport +
microsoft365 +
emailSecurity +
endpointSecurity +
backupRecovery +
networkSupport +
otherTools +
(monthlyAdminHours * adminHourlyRate)

currentAnnualSpend =
currentMonthlySpend * 12

### B. Overlap risk score
Create a weighted point model.

Suggested weights:
- more than one security tool = +2
- more than one backup tool = +2
- multiple support vendors = +2
- unsure whether all software is actively used = +2
- former employees left on paid licenses = +2
- manual onboarding = +1
- manual offboarding = +1
- unclear Microsoft 365 licensing = +2
- manual shared mailbox forwarding / handoff = +1

Return:
- Low overlap
- Moderate overlap
- High overlap

Suggested thresholds:
- 0 to 3 = Low overlap
- 4 to 8 = Moderate overlap
- 9+ = High overlap

### C. Compliance and control score
Create a weighted point model.

Suggested weights:
- no multi-factor authentication = +3
- onboarding not documented = +2
- offboarding not documented = +2
- Microsoft 365 administrative roles not limited / reviewed = +2
- unknown backup ownership = +2
- no restore testing = +2
- access changes not tracked consistently = +2
- compliance-conscious environment with weak controls = +2

Return:
- Strong baseline
- Needs review
- Priority gap

Suggested thresholds:
- 0 to 3 = Strong baseline
- 4 to 8 = Needs review
- 9+ = Priority gap

### D. Estimated savings opportunity
Use a realistic range, not fake exact precision.

Base savings guidance:
- Low overlap: 8% to 12% of currentMonthlySpend
- Moderate overlap: 12% to 20% of currentMonthlySpend
- High overlap: 18% to 30% of currentMonthlySpend

Then increase the range modestly if:
- Microsoft 365 licensing is unclear
- former employees remain on licenses
- manual onboarding and offboarding are present
- multiple vendors handle overlapping support
- duplicate backup or security tools are present

Show:
- Estimated potential monthly savings: $X to $Y
- Estimated potential annual savings: $A to $B

### E. Recommended next steps
Generate 3 to 5 recommendation items based on inputs.

Possible outputs:
- Review Microsoft 365 licensing
- Review overlapping security tools
- Standardize onboarding and offboarding
- Clarify vendor ownership
- Review backup responsibility and restore testing
- Review access control and administrative role practices
- Consolidate overlapping support tools
