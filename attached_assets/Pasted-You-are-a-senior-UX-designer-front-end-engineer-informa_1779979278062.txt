You are a senior UX designer, front-end engineer, information architecture reviewer, and quality assurance specialist.

I need you to do a full-site integration and navigation audit for the current **Grand Strand Ally** website and fix all routing, link behavior, and navigation consistency issues so the site is ready for production launch.

## Core objective
Make the site feel fully integrated, intentional, and reliable.

Specifically:
- audit every navigation item
- audit every button
- audit every text link
- audit every card link
- audit every call to action
- audit any homepage section links or anchor behavior
- make sure every link goes to the correct page or section
- make sure the Services and Pricing links go to the correct dedicated pages
- make sure all internal routing is consistent across desktop and mobile
- make sure there are no broken, confusing, duplicate, or mismatched destinations

## Important UX goal
The site should feel like one cohesive system, not a set of separate pages with inconsistent links.

I want:
- clean global navigation
- clear page-to-page routing
- predictable call-to-action behavior
- no dead ends
- no confusing mixed patterns
- no links that appear to go to a page but only jump to the wrong area
- no buttons that route inconsistently from one page to another

## Routing and navigation rules

### Global navigation
Header navigation should be consistent across all pages and should use proper page routes:
- Home → /
- Services → /services
- Pricing → /pricing
- Contact → /contact

Requirements:
- Services must always go to the full Services page
- Pricing must always go to the full Pricing page
- Contact must always go to the Contact page
- the logo should always return to the homepage
- mobile navigation must use the same destinations as desktop navigation
- active nav states should be handled correctly where appropriate

### Homepage CTA behavior
Audit all homepage buttons and links.

Rules:
- “Book a Free Cost Analysis” should consistently go to /contact unless there is a clearly intentional on-page form
- “See Services” should go to /services, not to a broken anchor or wrong page
- “Request Pricing” should go to /pricing or /contact depending on context, but this logic must be consistent and intentional
- if there are any “Learn more” service links, they should go to /services or the correct service section if sections exist
- if homepage uses section anchors, they must work correctly and scroll to the correct section IDs

### Page-level CTA behavior
On all internal pages:
- Services page CTA should lead to /contact or /pricing intentionally
- Pricing page CTA should lead to /contact for cost analysis
- Contact page should not end in a confusing circular CTA
- if the Contact page has a button currently linking to mailto, decide whether the primary action should be:
  1. submit the form
  2. stay on contact
  3. email directly as a secondary action
- the primary CTA on Contact should not feel redundant or broken

### Footer links
Audit all footer links and ensure they route correctly:
- Home
- Services
- Pricing
- Contact
- Privacy
- Terms

### Thank-you flow
If the contact form exists:
- successful form submission should redirect to /thank-you
- /thank-you should include a clear next-step message
- if form submission is stubbed, the logic should still be production-safe and easy to wire later

## Link behavior cleanup
Find and fix:
- incorrect href values
- section links pointing to missing IDs
- buttons pointing to the wrong page
- inconsistent use of anchors versus routes
- duplicated CTA patterns with different destinations
- mailto links being used as primary conversion flow where a page/form CTA is better
- any places where Services or Pricing go to the homepage or wrong section instead of their actual pages
- any mobile menu links that differ from desktop links
- any cards that look clickable but are not
- any clickable areas without clear affordance

## UX architecture requirements
Make the site navigation feel simpler and more intentional.

Use these rules:
- primary navigation = page routes
- section anchors only when clearly useful on the homepage
- service cards should have one predictable destination pattern
- primary CTA should be consistent sitewide
- avoid sending users in circles
- remove unnecessary duplicate pathways if they create confusion

## Technical implementation requirements
- use Next.js Link for internal routes
- use normal anchor tags only for external URLs, mailto, or true section anchors
- centralize route constants if useful
- make sure every href is valid
- clean up any route mismatches between desktop header, mobile menu, footer, hero buttons, card links, and CTA bands
- ensure keyboard accessibility and focus states for all links and buttons
- confirm the site builds cleanly after changes
