Build a complete “Generative Engine Optimization (GEO)” offering on gsally.com, including:

1. a public GEO service landing page
2. a lead capture / intake flow
3. an automated GEO audit workflow
4. an admin/report output structure
5. clear CTAs that connect the service page to the intake form and report workflow

IMPORTANT:
- Do not redesign the whole site.
- Match the existing visual system, typography, layout rhythm, CTA style, and brand tone already used on gsally.com.
- Keep the site’s current navigation, footer, forms, SEO setup, and deployment structure intact unless a change is required for this feature.
- Build this to be production-ready, not a mockup.
- If the codebase already has shared layout, button, section, card, form, and route components, reuse them.

GOAL OF THE OFFER:
Create a new service that helps companies improve how they appear in AI-generated search experiences and answer engines. The service should be positioned as a practical business offering, not hype. It should emphasize visibility, discoverability, citation-readiness, site structure, content quality, and ongoing improvement.

================================

================================

Create a new page/route for the offering. Preferred slug:

3. Who It’s For

- Not-a-fit bullets

================================
PART 3 — AUTOMATED GEO AUDIT WORKFLOW
================================

Build the first version of an automated GEO audit pipeline behind the intake form.

Objective:
When a form is submitted, create a structured audit job using the submitted website and business inputs. The workflow does not need to be fully enterprise-grade, but it must be real, organized, and extensible.

Required workflow behavior:

A. On successful form submission:
- store the submission
- create a unique audit job ID
- mark status as:
  - submitted
  - queued
  - processing
  - complete
  - failed

B. Audit modules to include in v1:
1. Crawl / page inventory
   - fetch homepage
   - try sitemap.xml
   - try robots.txt
   - gather key page URLs if discoverable
2. Technical checks
   - status code checks
   - sitemap presence
   - robots presence
   - canonical presence on homepage if detectable
   - title/meta extraction
   - indexability basics where possible
3. Business/entity extraction
   - business name
   - contact info if visible
   - service labels if visible
   - location mentions
4. Content architecture checks
   - existence of About, Services, Contact, FAQ, Case Studies / Testimonials, Pricing / Process pages if detectable
   - count/flag service pages if discoverable
   - identify likely missing high-intent page types
5. GEO content-gap checks
   Generate a rules-based assessment for missing page types like:
   - service pages
   - location/service pages
   - FAQs
   - comparisons
   - pricing / process pages
   - case studies
   - checklist / guide content
6. Internal structure checks
   - if key links are discoverable, note whether the site appears to have navigable structure
7. Output a GEO scorecard
   categories:
   - crawlability
   - technical readiness
   - entity clarity
   - service-page depth
   - trust/proof content
   - answer-engine content coverage
   - internal structure
   - conversion readiness

C. Recommendation engine
Based on the audit results, generate:
- top 5 highest-priority fixes
- top 5 content opportunities
- suggested next pages to build
- suggested implementation tier:
  - Quick Audit only
  - Foundation Build
  - Monthly GEO Growth

D. Report output
Generate a structured report object and a presentable human-readable summary.

At minimum produce:
- job metadata
- company info
- submitted inputs
- detected pages
- audit findings
- scorecard
- recommendations
- next-step CTA
