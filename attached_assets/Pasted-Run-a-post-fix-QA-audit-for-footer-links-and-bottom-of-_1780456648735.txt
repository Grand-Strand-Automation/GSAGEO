QA tasks:

2. Crawl and test the live site behavior for those links from multiple pages, especially from pages where the user is already scrolled near the bottom.
3. Confirm that clicking standard footer links:
   - changes route correctly
   - lands at the top of the destination page
   - does NOT preserve the previous page’s scroll position
4. Confirm that clicking anchor/hash links:
   - lands on the correct target section
   - does not overshoot or undershoot

6. Test behavior from desktop and mobile viewport sizes.
7. Check for broken, duplicated, incorrect, or inconsistent href values in the footer or bottom CTA components.
8. Check whether any footer link still points to plain /contact when it should point to /contact#contact-form.
9. Check whether any resource/article/service footer link points to an alternate or legacy URL instead of the intended canonical URL.
10. Check whether browser back/forward navigation still works reasonably after the scroll fix.
11. Confirm no console errors or route errors occur during navigation.
12. Confirm no footer links cause partial-page landing unless a hash anchor is intended.

Expected pass criteria:
- Every normal footer page link lands at the top of the correct page.
- Every intended hash link lands at the correct section.
- The contact form CTA lands with the form visible.
- No sticky-header overlap blocks the heading/section target.
- No broken footer links.
- No legacy or wrong hrefs in footer or bottom CTA links.
- No JS/router errors during navigation.

Rules:
